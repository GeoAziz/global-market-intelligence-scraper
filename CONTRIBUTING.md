# Contributing to GMIS

Thanks for wanting to contribute! Please follow these guidelines:

- Fork the repo and open a PR against `main`.
- Write tests for new features and run `npm test` locally.
- Keep PRs focused and include a short description of the change and rationale.
- Ensure no secrets are committed. Use `.env` for local testing and Apify actor env for secrets.

Code style
----------
- Use consistent indentation and small functions.

Review
------
- At least one approving review is required before merge.
